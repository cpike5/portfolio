HTML Structure 
<!-- Mobile Header -->
<header class="mobile-header">
    <button class="mobile-menu-button" aria-label="Menu">
        <svg class="menu-icon">...</svg>
    </button>
    <div class="mobile-title">cpike.ca</div>
</header>

<!-- Overlay -->
<div class="mobile-menu-overlay" id="overlay"></div>

<!-- Menu -->
<nav class="mobile-menu" id="menu">
    <div class="mobile-menu-header">
        <span class="mobile-menu-title">Menu</span>
        <button class="mobile-menu-close" aria-label="Close menu">
            <svg class="close-icon">...</svg>
        </button>
    </div>
    <ul class="mobile-nav-links">
        <li><a href="#home" class="mobile-nav-link active">Home</a></li>
        <li><a href="#about" class="mobile-nav-link">About</a></li>
        <!-- more links -->
    </ul>
</nav>
JavaScript (for reference) 
// Open menu
menuButton.addEventListener('click', () => {
    menu.classList.add('active');
    overlay.classList.add('active');
    document.body.style.overflow = 'hidden';
});

// Close menu
closeButton.addEventListener('click', closeMenu);
overlay.addEventListener('click', closeMenu);

function closeMenu() {
    menu.classList.remove('active');
    overlay.classList.remove('active');
    document.body.style.overflow = '';
}
Blazor Component Props 
Proposed Props:

NavItems (List<NavItem>)
  - Text: string (display text)
  - Href: string (link/route)
  - IsActive: bool (active state)

IsOpen (bool)
  - Controls menu visibility

SiteTitle (string)
  - Header title text (default: "cpike.ca")

OnToggle (EventCallback<bool>)
  - Menu open/close event

OnNavigate (EventCallback<string>)
  - Navigation click handler

Implementation Notes

Behavior:
• Menu slides from top: translateY(-100%) → translateY(0)
• Overlay fades in with backdrop blur
• Body scroll locked when menu open
• Auto-close on navigation
• Close on overlay click

Accessibility:
• Aria labels on icon buttons
• Keyboard navigable (Tab, Enter, Escape)
• Focus trap within open menu
• Screen reader friendly

Responsive:
• Shown only on mobile (<768px)
• Full viewport height
• Large touch targets (1.25rem padding)
• Centered vertical navigation
