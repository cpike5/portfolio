CSS Classes & Styling 
Classes:
  .projects-grid             - Grid container
  .project-card              - Individual project card
  .project-image             - Project screenshot
  .project-image-placeholder - Placeholder when no image
  .project-title             - Project title
  .project-description       - Project description
  .project-link              - GitHub link

Grid Layout:
  • Display: grid
  • Template: repeat(auto-fit, minmax(300px, 1fr))
  • Gap: 2rem
  • Responsive: auto-fit adjusts columns

Project Card:
  • Display: flex, flex-direction: column
  • Padding: 2rem
  • Border: 1px solid rgba(183, 65, 254, 0.2)
  • Border-radius: 8px
  • Background: gradient (primary with low opacity)
  • Top accent: 3px gradient line

Image Styling:
  • Width: calc(100% - 1rem) (inset from edges)
  • Height: 200px
  • Object-fit: cover
  • Border-radius: 8px
  • Margin: 0 auto 1.5rem

Placeholder:
  • Same dimensions as image
  • Gradient background with pattern
  • Centered "Screenshot" text

Hover Effects:
  • Card: translateY(-5px)
  • Border: solid primary color
  • Background: intensified gradient
  • Title: translateX(5px)
  • Link gap: increases

Responsive:
  • Desktop: 2-3 columns (auto-fit)
  • Tablet (≤768px): 1 column
  • Mobile (≤480px): 1 column, reduced padding

Styling Reference:
  styles.css:529-670
Blazor Component Props 
Proposed Props:

Projects (List<Project>) [Required]
  - Collection of projects to display

Project Model:
  - Title: string
  - Description: string
  - ImageUrl: string (optional)
  - ImageAlt: string
  - LinkUrl: string
  - LinkText: string (default: "View on GitHub →")

GridGap (string)
  - Gap between cards (default: "2rem")

MinCardWidth (string)
  - Minimum card width (default: "300px")

CssClass (string)
  - Additional CSS classes

Implementation Notes

Grid Behavior:
• Auto-fit creates responsive columns automatically
• Minimum card width: 300px
• Cards expand to fill available space
• Stacks to single column on mobile

Card Layout:
• Flexbox column layout
• Image/placeholder at top
• Title and description in middle
• Link pushed to bottom (margin-top: auto)

Images:
• Fixed height (200px) for consistency
• Object-fit: cover maintains aspect ratio
• Placeholder shown when no image available
• Consider lazy loading for performance

Interaction:
• Card lifts on hover (5px)
• Title shifts right on card hover
• Link arrow gap increases on hover
• Smooth transitions throughout

Content Guidelines:
• Keep descriptions concise (2-3 sentences)
• Use descriptive alt text for images
• Provide meaningful project titles
• Include working links to repositories

Accessibility:
• Semantic HTML structure
• Proper heading hierarchy
• Descriptive link text
• Image alt attributes
• Focus states on interactive elements
