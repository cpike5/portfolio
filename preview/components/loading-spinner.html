CSS Classes & Styling 
Classes:
  .submit-loader      - Small spinner (16px) for buttons
  .overlay-spinner    - Large spinner (48px) for overlays

Button Spinner (.submit-loader):
  • Display: none (shown when button has .loading)
  • Width: 16px
  • Height: 16px
  • Border: 2px solid transparent
  • Border-top-color: var(--color-primary)
  • Border-radius: 50%
  • Animation: spin 0.8s linear infinite
  • Position: absolute, centered (top 50%, left 50%)

Overlay Spinner (.overlay-spinner):
  • Width: 48px
  • Height: 48px
  • Border: 3px solid rgba(183, 65, 254, 0.2)
  • Border-top-color: var(--color-primary)
  • Border-radius: 50%
  • Animation: overlaySpinAnimation 0.8s linear infinite

Spin Animation:
  @keyframes spin {
    to {
      transform: translate(-50%, -50%) rotate(360deg);
    }
  }

  @keyframes overlaySpinAnimation {
    to {
      transform: rotate(360deg);
    }
  }

Button Loading State:
  .form-submit.loading .submit-text {
    opacity: 0;
  }

  .form-submit.loading .submit-loader {
    display: block;
  }

Implementation Notes

Animation:
• 360-degree rotation
• Linear timing (constant speed)
• 0.8-second duration
• Infinite loop

Design:
• Circular border with transparent sections
• Only top border visible (creates arc)
• Rotation creates spinning effect
• Smooth, GPU-accelerated animation

Usage Contexts:
• Button spinners: 16px, hidden by default
• Overlay spinners: 48px, shown during async ops
• Inline spinners: 24px, for in-content loading
• Custom sizes as needed

Color Customization:
• Border-top-color controls visible arc
• Border-color controls track (usually transparent or subtle)
• Match to primary color for consistency
• Adjust contrast for different backgrounds

Accessibility:
• Add aria-label for context
• Announce loading state to screen readers
• Pair with text labels when possible
• Don't rely solely on spinner for feedback

Performance:
• Use transform for animation (GPU-accelerated)
• Avoid animating layout properties
• Remove from DOM when not needed
• Consider reducing motion preference
